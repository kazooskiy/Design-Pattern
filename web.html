1. An explanation of what a design pattern is

A design pattern is a format where the javascript is organized so it is functional and easy to read.

2. An explanation of each respective pattern in terms of what it is and why it is used.

Creational- Singliton: A class where only one instance(copy) can exist, and provide a global point to access. This is used because Singletom reduces the use of global variables in order to prevent name collisions.
Structural- Facade: It provides an interface to a set of another interface in a subsystem. It makes the subsystem eay to use. Using the Facade Pattern only shows the presentation layer, and hides the business objects and their interactions. It is also used to refactor codes.
Behavioral- Memento: Provide temporary storage and restoration of an object in Javascript. It is used to take snapshot of an objects state so changes can be altered easily.
